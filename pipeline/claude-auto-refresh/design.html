Notes on this direction
  1. Zero new CSS, zero new layout: both new states are copy branches rendered with the shipped data-quality note (.stale-note) in its existing slot between the gauges and the pacing callout. The age pill, gauges, pacing rows, tiles, mix, and trends are untouched in every state; Codex is untouched.
  2. One note at a time: the server reports exactly one reason (failing > disabled > stale-reading > no-statusline-reading). The new notes open with the staleness sentence themselves, so a failing or disabled state never stacks a second stale note.
  3. Silence is health: when auto-refresh is working, nothing in the UI says so — the fresh age is the only evidence. No status indicator was invented for the healthy path.
  4. Text first, cause in plain words: each note states what's happening ("Auto-refresh is failing" / "Auto-refresh is off"), why, and the fix — never a bare error code. The four failure causes each get their own sentence (see design-spec.md copy table).
  5. Live ages: the note's "updated 47m ago" re-derives from capturedAt on the existing 1-second render tick, same as the stale note today; tabular-nums keeps it from jittering.
  6. Omitted for focus: activity tiles, token mix, and trends render below the pacing callout exactly as today in every state — the diagnostics never touch them.
