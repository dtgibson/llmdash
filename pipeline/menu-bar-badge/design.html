Which tool is tight? — a compact binding-tool cue (adopted)

The glyph shows the tightest number but not which tool it belongs to. A single-letter cue after the mark answers that at a glance without clutter: C = Claude, X = codeX. One extra monospace character, xbar-safe (text floor; an SF Symbol is optional SwiftBar polish). It reads ▪C 46% — "llmdash · Claude · 46% left." Compared side by side below; adopted because it stays clean.

Verdict: adopted. On a narrow bar the cue stays a single clean token (▪C 46% / ▪X 12%), never busy, and it carries genuinely useful information — which tool is about to throttle you. The cue is muted (it's a label, not the number), so the eye still lands on the percentage first. It rides the same honesty treatment as the number: dimmed slightly on aging, present on stale. It is omitted in the no-reading and offline states (there's no binding tool to name).

The five honesty states

The leading ▪ is the stable llmdash mark — it never changes, so the badge is always recognizable. The tool cue (C/X), the number, the trailing marker, and the color carry the state. Two rules are structural: an aging/stale reading is never shown as confidently fresh, and offline / no-reading never show a number.

Glyph symbol + color — the binding spec (OQ-03 resolved)

This is what the Engineer implements verbatim. Colors are named against the design system's status tokens; the menu-bar strip uses lifted variants of the same hue for contrast on a dark bar. The tool cue (C Claude / X codeX) is muted and rides the glyph's state; it is omitted in no-reading and offline (no binding tool to name).

State Glyph Color token The rule it enforces
fresh ▪ C 46% status of the % (good/warn/crit) Plain, confident. The number's own status color shows through; muted tool cue.
aging ▪ C 78%· status hue kept, ~0.82 opacity Keeps its status color (you still see how much) + trailing · and a hair of de-emphasis. The · is the load-bearing marker — reads in a monochrome bar.
stale ▪ C 78% ⚠ warn / #f0a94b Number tinted amber + trailing ⚠. Present but flagged, never confident.
no-reading ▪ — muted / #9b9ea6 A dash, never a number, no tool cue. Dropdown explains why per tool.
offline ▪ llmdash ⚠ muted / #8b8b8b Wordmark + slash, dimmed. Never a number. "No server" ≠ "lots of headroom".
xbar-safe floor + SwiftBar polish. Every state above is carried by text/emoji + color= — nothing here depends on a SwiftBar-only SF Symbol, so the badge stays honest on xbar too. On SwiftBar, the leading ▪ mark may optionally be swapped for a monochrome sfimage= template glyph as polish; the text mark is the floor and always ships.  ·  The trailing marker on stale (⚠) and the dimming on aging are the honesty signals; they are never color-alone — the marker/dash/dim reads even in a monochrome menu bar.  ·  Countdowns (resets 3h 12m) format exactly as the dashboard's fmtDur; they reflect the moment the plugin last ran and need not tick between runs.
