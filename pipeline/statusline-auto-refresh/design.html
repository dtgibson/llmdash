statusline-auto-refresh — reading-age treatment

The Claude block's reading age, in four states. One statusline capture feeds both gauges, so one flag serves the block: the age stays where it lives today (the tool header) and picks up a status pill as it crosses each band — fresh ≤ 5m · aging 5–10m · stale > 10m (10m is 2× LLMDASH_CLAUDE_MAX_AGE_MS, one knob). A stale reading is flagged, never blanked: the gauges keep rendering the last capture, and the note below them states the age and names the remedy.

Notes on this direction
  1. What's unchanged: activity tiles, token mix, and trends render below the pacing callout exactly as today in every state. The freshness treatment never touches them — activity derives from local session logs, not the statusline reading. Omitted here for focus. Codex is untouched (not retrofitted this feature).
  2. Live ages: every age here ("updated 7m ago", the note's "updated 1h 24m ago") re-derives from capturedAt on the existing 1-second render tick, so the band and copy stay current between fetches.
  3. Age granularity: at hour scale the age shows hours + minutes ("updated 1h 24m ago", not "updated 1h ago") — a small fmtAge extension, so a stale reading states its age honestly.
  4. Text first: staleness is carried by the words "stale" (pill) and "Stale reading" (note) before any color — the tints reuse the dashboard's established warn/crit pill vocabulary (NFR-08).
  5. Bands: fresh ≤ 5m · aging 5–10m · stale > 10m; the stale band is always 2× the LLMDASH_CLAUDE_MAX_AGE_MS threshold (default 300000 = 5m) — one knob, two thresholds.
