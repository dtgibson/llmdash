llmdash · menu-bar service controls · design draft

Install, remove, and uninstall — all from the badge, never the terminal.

Two lifecycle controls join the dropdown, riding the same ＋ / － / ☰ action grammar the shipped badge already uses. A state-aware service toggle reflects the real launchd state — － Remove the local service · running when it's up, ＋ Install the local service when it isn't. An Uninstall llmdash… submenu offers two tiers, and a complete uninstall enumerates every artifact before it touches anything. Your usage-history database is preserved by default — the one thing here that can't be rebuilt.

Draft · first direction · react and iterate

The state-aware label, verbatim. The service item reads the real launchd state each render — never a faked checkmark — and there are three states:
  • ＋ Install the local service — when not installed (no plist on disk). A light confirm; it regenerates the plist with fresh absolute paths and loads it.
  • － Remove the local service · running — when running (loaded into the user domain). The · running suffix is a dim mono locator, not a color.
  • － Remove the local service · stopped — when the plist is present but unloaded. Honest "stopped," not "not installed"; Remove still deletes the plist.
Both items sit in the same slot in both modes — a fresh single-host or a monitoring-station Mac reaches them identically.

Uninstall llmdash… — two tiers, one submenu

Uninstall llmdash… opens a submenu with two clearly-separated tiers: a light one that only takes the badge off the bar, and a heavy one that tears the whole install down (behind an enumerated confirm). The two are never one click apart by accident — the complete one carries its own … and its own gate.

Two tiers, two very different stakes:
  • Remove the menu-bar badge only — reverses --setup-badge and nothing more. The marker-gated SwiftBar wrapper is removed; the service keeps running, the checkout and your data are untouched. The badge just disappears from the bar on the next refresh. A file in SwiftBar's dir without our marker is a file of yours — it's left alone with an honest message.
  • Uninstall llmdash completely… — the trailing … signals it opens a dialog first. It enumerates every artifact (next section) and only acts on confirm, then asks separately about your history.
SwiftBar is never removed. llmdash never runs brew uninstall --cask swiftbar — SwiftBar is your app, a prerequisite we don't manage. Both the enumeration and the post-uninstall message point you to that command if you want it gone; the action itself never touches it.

The enumerated confirmation — everything named before anything is touched

Choosing Uninstall llmdash completely… pops a fixed-literal osascript display dialog that lists every artifact it will remove, by name and path, before it removes a single one. The resolved checkout path is embedded (escaped, never re-fed as script). The one thing that can't be rebuilt — your snapshot history — is called out as preserved by default. The default button is Cancel; Uninstall must be chosen deliberately.

Cancel is the default — nothing happens until you choose Uninstall. Uninstall Cancel

Cancel changes nothing — no service touched, no file removed. Only Uninstall proceeds, and only to step 2 (your history question) before anything is deleted.

Why enumerate first? A complete uninstall is irreversible for everything but the database. Listing each artifact — by name and real path — before acting means you approve the exact scope, never a vague "are you sure?". Every path here is marker-gated: the plist only if it's the com.llmdash.dashboard label; the statusline only if it points at this checkout; the wrapper only if it carries our marker; the checkout only the resolved ~/llmdash. A second llmdash install, or a file of yours, is left untouched — and if any step can't complete, the result says exactly what did not happen, never claims a removal that didn't occur.

The two remaining dialogs — your history, and removing the service

Two more dialogs complete the picture. Left: the data opt-in, which only appears after you've confirmed the uninstall — the safe choice (Keep my history) is the default button, and the destructive one names its irreversibility. Right: the standalone Remove the local service confirm, which names the consequence — this Mac stops monitoring locally, but the badge keeps working off remote hosts.

the service-remove confirm
▪
Remove the local llmdash service?
This Mac will stop running its own local monitor — the launchd agent is unloaded and its plist deleted. If this badge watches remote machines, it keeps working off those; only the local reading stops. You can re-install the service from this menu any time.
Remove the service Cancel

Cancel is the default. Install gets a lighter confirm of the same shape ("Install the local llmdash service on this Mac?") — every mutation is confirmed, but a non-destructive one reads lighter.

One consistent discipline across all three dialogs: the default button is always the safe choice (Cancel, or Keep my history) — so a reflexive Return key never destroys anything. The destructive button is always plainly labeled with the consequence (Uninstall, Delete history too, Remove the service), never a bare "OK." Every dialog is a fixed literal invoked with no shell; the only dynamic value — the resolved checkout path — is escaped into the string, never re-run as script.

Reuse over reinvention. The dropdown chrome, the ＋ / － / ☰ action grammar, the submenu affordance, and the .osa-dialog macOS-dialog styling are all lifted from the shipped multi-host badge — the service item and Uninstall submenu are new rows in an existing vocabulary, not new chrome.  ·  No new color semantics: the enumerate dialog's preserve-by-default note reuses the good tint, the data opt-in reuses warn, the destructive buttons carry the crit hue — all existing design-system tokens.  ·  OS-native where it belongs: the dialogs match macOS display dialog conventions (icon, title, body, two buttons, default on the safe choice) because they are real osascript dialogs, not llmdash surfaces.  ·  Honesty floor: live state never faked, every artifact enumerated before acting, the irreplaceable database preserved by default, SwiftBar never removed.
